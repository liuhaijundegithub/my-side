### 批量导入使用指南

#### 1、基本概念

我们将批量导入分为了五个部分，分别为五个阶段

- 选择文件
- 预览数据
- 入库结果
- 历史记录
- 历史记录详情

对应了steps里的五个组件

对应了service里的step1 - step4，其中的历史记录详情页面不需要新接口（调用入库结果的接口）

#### 2、基本逻辑

- 每个阶段均没有涉及到路由切换。都集中的在 `BatchImport.tsx` 中管理 。使用的 `model.current` 来控制每一个阶段的显示
- 所有页面使用到的iconfont 都是出自于 `@ant-design/icons` 中，可按需替换
- `const.ts` 中定义了批量导入的业务类型，注意，这个要和businessType区分开。这是要作为参数传给后端的，但是后端定义的参数名称可能不一定，这里先使用 `excelType` 占位
- `types.d.ts` 中定义了 BatchImport的参数类型

#### 3、使用

> 因为是个半成品，所以需要你

- `excelType`
  - 确定后端的业务类型是怎么定义的，即 `excelType` 这个字段后端是怎么定义的，你需要将 `BatchImportProps` 中的这个参数修改成与后端一致
  - `service/step1.ts` 中上传excel 地址接口 是需要这个参数，需要与后端一致
  - `service/step4.ts` 中的接口同上
  - 最好检查一下service中的几个接口与后端保持一致
  - 我会将这个参数透传到需要它的接口上，如果有遗漏的地方需要你仔细核对一下四个step的接口定义与对应的后端文档一致
- 模板下载
  - 我们提供一个getTemplate方法，我们会把当前路由上的所有参数注入这个回调方法中。在你需要使用的时候去使用。
- columns
  - 提供了columns参数和getColumns方法，二者必须有一个，getColumns的优先级更高。如果提供了这个方法，则会在一开始就执行这个方法就执行，并且透传到各个组件中去
- 样式
  - 样式风格基本沿用了云校的风格
  - 按需修改1
  - components/header 的样式需要重写一下，依据业务设计图，我们内置了一个【历史记录】按钮和一个【返回按钮】
- @pages/import中提供了一个demo页面 localhost:1080/import 可访问，用了基础数据的student导入作为了全流程
